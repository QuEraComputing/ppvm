# generated/codes/

`stim gen` sweep over fault-tolerance circuits. Generated by:

```bash
cd crates/ppvm-stim/tests/regen-stim
uv run regen-stim codes
```

Sweep axes:
- `--code` ∈ {`surface_code`, `repetition_code`, `color_code`}
- `--task` ∈ all task variants (some emit phase-1-unsupported instructions
  like `MX`, `RX`, `C_XYZ` and land in `../unsupported/` instead)
- `--distance` ∈ {3, 5, 7}
- `--rounds` ∈ {1, 3, 5}
- noise: noiseless, p=0.001, p=0.01

Distance ≤ 5 fixtures use `num_shots=256`; distance = 7 uses `num_shots=64`.

Circuits whose maximum qubit index is ≥ 64 are skipped: the cargo harness
in `tests/stim_corpus.rs` uses a fixed 64-qubit tableau (storage `<8>`),
and bumping the budget requires changing the storage parameter and
regenerating every committed `ppvm_bit_means`.

The `regen-stim codes` script auto-routes fixtures whose generated circuit
contains a phase-1-unsupported instruction (e.g. `MR` for some tasks,
`SWAP`, `MPP`, `MX`, `RX`, `C_XYZ`) into `../unsupported/`. Detection walks
the parsed circuit (descending into REPEAT bodies) so it matches ppvm's
normalizer order — a regex over the source text would miss instructions
inside indented REPEAT blocks.
