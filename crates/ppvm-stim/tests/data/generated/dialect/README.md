# generated/dialect/

ppvm-specific dialect instructions: `I[R_X(theta=…)]`, `I[R_Y(theta=…)]`,
`I[R_Z(theta=…)]`, `I[U3(theta=…, phi=…, lambda=…)]`, `S[T]`, `S_DAG[T]`.

Stim cannot simulate these — there is no oracle here. Deterministic-mode
fixtures are hand-derivable (e.g. `R_X(π)` flips |0⟩ → |1⟩, `T·T† = I`);
distribution-mode fixtures record ppvm's output directly to lock down
regression behavior.

This category catches ppvm dialect bugs only by detecting drift from
previously-recorded ppvm output, not by cross-checking against another
simulator. If you change the rotation gate semantics in
`crates/ppvm-tableau/src/gates/`, expect these fixtures to break and
`regen-stim refresh <path>` them after verifying the new behavior is correct.

Generated by:

```bash
cd crates/ppvm-stim/tests/regen-stim
uv run regen-stim dialect
```
