Core runtime for the ppvm quantum-circuit simulator.

ppvm-runtime defines the low-level types every ppvm backend uses: single-qubit char::Pauli symbols, packed word::PauliWord strings, phased and lossy variants, the sum::PauliSum dictionary of Pauli strings to coefficients, and the traits hierarchy of gates, measurements, and noise channels.

The crate is generic over a config::Config bundle that fixes storage, coefficient type, hashing, and truncation strategy at compile time. Higher-level crates (ppvm-tableau, ppvm-sym) build on top of these types.

§Quick example

Compute <ZZ> after the GHZ-preparation circuit H(0); CNOT(0, 1) by propagating ZZ backwards in the Heisenberg picture:

use ppvm_runtime::prelude::*;

let mut state: PauliSum<config::indexmap::ByteFxHashF64<1>> =
    PauliSum::builder().n_qubits(2).build();
state += ("ZZ", 1.0);

// Circuit is H(0); CNOT(0, 1) — apply in reverse for Heisenberg propagation.
state.cnot(0, 1);
state.h(0);

// ZZ has propagated to a single term.
assert_eq!(state.len(), 1);

Modules§

char
Single-qubit Pauli alphabet (I, X, Y, Z, and a loss marker L).
config
Pre-bundled Config implementations and the trait itself, which fixes storage / coefficient / hashing / strategy at compile time.
loss
Loss-aware Pauli words that carry a parallel bitmap of lost qubits.
pattern
Pauli “patterns” — sets of compatible Pauli words used for masking and selective gate application.
phase
Phased Pauli words: a Pauli word paired with one of the four fourth-roots of unity ±1, ±i.
prelude
Convenience re-exports of the most common types and traits.
strategy
Truncation strategies that decide when low-weight or low-magnitude terms are dropped from a PauliSum.
sum
The PauliSum type and its builder.
traits
Gate, measurement, and noise-channel traits shared across backends.
word
Packed PauliWord — a fixed-width Pauli string stored as u8-packed Pauli digits.
