Expand description

Combined in-place modify + insert pattern used to express branching gates (where one input entry can produce several output entries).

Required Methods§

Source

fn map_insert<F>(&mut self, dest: &mut Self, f: F)
where F: Fn(&W, &mut V) -> Option<(W, V)> + Sync + Send,

Modify each existing entry in place; if f returns Some((k', v')), also insert that new entry into dest.

Source

fn map_insert_vec<F>(&mut self, dest: &mut Vec<(W, V)>, f: F)
where F: Fn(&W, &mut V) -> Option<(W, V)> + Sync + Send,

Like map_insert but appends produced entries into a plain Vec (a push per entry, no hashing) instead of a map. The caller merges the buffer into the destination map afterwards, avoiding a second hashmap probe per produced entry.

Source

fn map_insert_multiple<F>(&mut self, dest: &mut Self, f: F)
where F: Fn(&W, &mut V) -> Option<Vec<(W, V)>> + Sync + Send,

Same as map_insert but f may return a Vec of new entries to be inserted into dest.

Dyn Compatibility§

This trait is not dyn compatible.

In older versions of Rust, dyn compatibility was called "object safety", so this trait is not object safe.
